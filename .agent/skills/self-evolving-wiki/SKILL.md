# 🧬 Self-Evolving Wiki Skill

## Goal
코딩 작업 중 발생하는 **학습 가능한 순간**을 포착하고, 체계적으로 위키에 축적하여 미래의 자신(AI + 대표님)이 같은 문제를 빠르게 해결하도록 만드는 자가발전 시스템.

---

## 🔍 학습 가능한 순간 (Learnable Moments) 감지

다음 상황이 발생하면 위키에 기록할지 판단한다:

### 자동 기록 대상 (무조건 기록)
1. **버그 해결**: 1시간 이상 디버깅한 문제의 원인과 해결법
2. **환경 설정**: 새로운 도구/라이브러리 설정 과정 (재현 가능한 단계)
3. **API 패턴**: 외부 API 호출 시 발견한 주의점 (rate limit, 인코딩 등)
4. **성능 개선**: 속도/메모리 개선 시 before/after 측정값

### 선택적 기록 대상 (대표님 확인 후)
5. **아키텍처 결정**: 왜 A 방식 대신 B를 선택했는지
6. **코드 패턴**: 자주 반복되는 코딩 패턴 (유틸 함수화 가능)
7. **실패 기록**: 시도했으나 실패한 접근법 (미래의 삽질 방지)

## 📝 기록 형식

위키 문서는 `.secondbrain/10_Wiki/🛠️ Projects/` 또는 `.secondbrain/10_Wiki/` 하위에 저장한다.

### 템플릿: 코딩 TIL (Today I Learned)

```markdown
# [제목] - YYYY-MM-DD

## 문제
[무엇이 문제였는지 1-2줄]

## 원인
[왜 발생했는지]

## 해결
[어떻게 해결했는지 — 코드 포함]

## 교훈
[미래에 같은 상황에서 바로 적용할 수 있는 원칙]

## 태그
`#python` `#fastapi` `#shorts-automation` `#debug`
```

### 템플릿: 기술 레시피

```markdown
# [기술/도구명] 레시피

## 언제 쓰는가
[이 기술이 필요한 상황]

## 설정 방법
[단계별 가이드]

## 주의점
[흔한 실수와 회피법]

## 예제 코드
[실제 프로젝트에서 사용한 코드]
```

## 🔄 자가발전 루프

```
코딩 작업 시작
    ↓
[1] AGENTS.md 읽기 (프로젝트 컨텍스트)
    ↓
[2] 기존 위키 검색 (관련 문서 있는지)
    ↓
[3] 작업 수행
    ↓
[4] 학습 가능한 순간 감지?
    ├── YES → 위키에 기록 (TIL 또는 레시피)
    │         ↓
    │   index.md에 링크 추가
    │         ↓
    │   AGENTS.md 업데이트 필요? → 업데이트
    └── NO → 다음 작업으로
```

## 📊 위키 건강도 체크 (월 1회 권장)

대표님이 `/schedule` 명령으로 월 1회 위키 정리를 설정할 수 있다:

1. **중복 검사**: 비슷한 주제의 문서가 있으면 병합 제안
2. **최신성 검사**: 3개월 이상 업데이트 없는 문서에 `[REVIEW NEEDED]` 태그
3. **연결성 검사**: index.md에 누락된 문서 링크 추가
4. **AGENTS.md 동기화**: 위키에서 발견한 새 패턴을 AGENTS.md에 반영
