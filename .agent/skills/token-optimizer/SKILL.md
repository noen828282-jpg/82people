# 🎯 Token Optimizer Skill

## Goal
AI 에이전트의 토큰 사용량을 최소화하면서 코딩 품질을 유지하는 것.
**모든 코딩 작업에서 이 지침을 자동으로 적용한다.**

---

## Phase 1: 탐색 (Exploration) — 검색 우선 전략

### 규칙 1: 절대 파일 전체를 먼저 보지 않는다
- ❌ `view_file`로 800줄짜리 파일 전체를 보는 것은 금지
- ✅ 먼저 `grep_search`로 관련 키워드/함수명을 검색
- ✅ 검색 결과에서 해당 줄 번호를 확인한 후, `view_file`의 `StartLine/EndLine`로 필요한 범위만 조회

### 규칙 2: 디렉토리 탐색은 점진적으로
- ❌ 프로젝트 전체를 재귀적으로 탐색하지 않는다
- ✅ `AGENTS.md`의 프로젝트 구조를 먼저 참조
- ✅ 필요한 디렉토리만 1단계씩 `list_dir`로 탐색

### 규칙 3: 이미 아는 것은 다시 묻지 않는다
- `AGENTS.md`에 기술 스택, 프로젝트 구조, 코딩 규칙이 정의되어 있음
- Knowledge Items에 저장된 패턴이 있으면 그것을 먼저 사용
- 이전 대화에서 이미 파악한 정보를 반복 조회하지 않음

---

## Phase 2: 편집 (Editing) — 최소 범위 수정

### 규칙 4: 수정은 항상 최소 범위로
- ❌ `write_to_file`로 파일 전체를 덮어쓰는 것은 **마지막 수단**
- ✅ 단일 연속 블록 수정: `replace_file_content` 사용
- ✅ 여러 비연속 블록 수정: `multi_replace_file_content` 사용
- ✅ `TargetContent`는 정확히 변경할 줄만 포함 (전체 파일 아님)

### 규칙 5: 한 파일에 대해 병렬 수정 도구를 호출하지 않는다
- 같은 파일에 대한 여러 수정은 `multi_replace_file_content`의 `ReplacementChunks`로 묶기
- 다른 파일에 대한 독립적 수정은 병렬 호출 OK

---

## Phase 3: 실행 (Execution) — 병렬화 & 캐싱

### 규칙 6: 독립적인 작업은 병렬로
- 서로 의존성이 없는 도구 호출은 같은 `<function_calls>` 블록에 묶어서 병렬 실행
- 예: 파일 A 검색 + 파일 B 검색 → 동시에 호출

### 규칙 7: 명령어 실행 시간 예측
- 빠르게 끝날 명령: `WaitMsBeforeAsync`를 충분히 크게 (동기 실행)
- 오래 걸릴 명령: 작게 설정하고 비동기로 처리
- 실패할 수 있는 명령: 500ms 정도로 설정

---

## Phase 4: 응답 (Response) — 간결하게

### 규칙 8: 응답은 핵심만
- 아티팩트에 상세 내용을 담고, 응답에서는 요약만 제공
- 코드를 반복 인용하지 않음 (파일 링크로 대체)
- "다음 단계" 제안은 3개 이내로

### 규칙 9: 이미 보여준 코드를 다시 출력하지 않는다
- 파일을 수정한 후 전체 파일을 다시 보여주지 않음
- 변경된 부분만 diff 형태로 설명

## 토큰 절약 체크리스트

작업 시작 전 이 체크리스트를 내부적으로 확인:

- [ ] AGENTS.md를 읽었는가? (프로젝트 정보 반복 조회 방지)
- [ ] Knowledge Items에 관련 패턴이 있는가?
- [ ] grep_search로 먼저 위치를 파악했는가?
- [ ] view_file에 StartLine/EndLine을 지정했는가?
- [ ] 병렬 호출 가능한 작업을 묶었는가?
- [ ] 수정 범위가 최소인가?
- [ ] 응답이 간결한가?

## 위반 시나리오 (Anti-patterns)

| ❌ 비효율적 | ✅ 효율적 |
|-------------|-----------|
| 668줄 파일 전체 `view_file` | `grep_search` → 해당 범위만 조회 |
| `write_to_file`로 전체 덮어쓰기 | `replace_file_content`로 변경 부분만 수정 |
| 같은 프로젝트 구조를 매번 `list_dir`로 탐색 | `AGENTS.md`의 구조 섹션 참조 |
| 검색 결과를 그대로 응답에 복사 | 파일 링크와 줄 번호만 제공 |
| 독립적 작업을 순차 실행 | 병렬 `<function_calls>` 호출 |
