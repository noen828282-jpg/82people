#1 압도적 1위 가장 많이 쓰는 기능
HOT 🧩

VS Code 확장 프로그램 (.vsix) 호환

Anti-Gravity의 핵심 강점. VS Code용으로 만들어진 .vsix 확장 프로그램을 그대로 설치해서 쓸 수 있습니다. Connect AI, ESLint, Prettier, GitHub Copilot 등 VS Code 생태계를 그대로 활용합니다.

  • → Connect AI 설치: 사이드바 AI 에이전트 + 가상 사무실
  • → ESLint / Prettier / GitLens 등 기존 VS Code 확장 그대로
  • → .vsix 파일 직접 설치 (커맨드라인 또는 드래그앤드롭)
code --install-extension connect-ai-lab-2.89.153.vsix --force
# extensions 폴더: ~/.antigravity/extensions/
⚠ 단점

VS Code와 100% 호환은 아닙니다. 일부 VS Code 전용 API를 쓰는 확장은 동작 안 하거나 버그납니다. 확장 업데이트 시 Anti-Gravity 버전과 충돌하는 경우도 있습니다. Marketplace에서 직접 설치가 안 되고 .vsix를 직접 받아야 하는 경우도 있습니다.

✓ 해결책

확장이 안 되면 VS Code에서 먼저 테스트하고 Anti-Gravity로 이동하세요. 버전 고정이 필요하면 --force로 재설치합니다. extensions 폴더에서 이전 버전 .vsix를 직접 삭제해서 충돌을 없앱니다.

.vsix 설치 VS Code 생태계 Connect AI 호환 ~/.antigravity/extensions/
#2 오프라인 & 로컬 실행
HOT 📡

완전 오프라인 작동

인터넷 없이도 에디터 전체가 동작합니다. 로컬 LLM(LM Studio, Ollama)과 연결하면 AI 기능도 오프라인으로 씁니다. 비행기, 지하, 와이파이 없는 시연 환경에서 완벽합니다.

  • → 와이파이 끊고 AI 채팅 — 정상 동작
  • → 로컬 LLM + Connect AI = 완전 오프라인 AI 개발
  • → Git, 터미널, 디버거 전부 오프라인 동작
⚠ 단점

Google 계정 동기화, Extensions Marketplace, GitHub 연동은 당연히 인터넷이 필요합니다. 로컬 LLM은 GPU/RAM이 부족하면 느리거나 OOM(메모리 부족) 으로 죽습니다.

✓ 해결책

오프라인 시연 전에 온라인 상태에서 확장/모델을 미리 다운받아 두세요. RAM 16GB 이상이면 7B 모델 안정적. 부족하면 3B 모델(llama-3.2-3b)로 낮추세요.

⚡

로컬 LLM 브릿지 (:4825)

HTTP 브릿지가 포트 4825에서 실행되어 LM Studio(:1234) 또는 Ollama(:11434)와 연결합니다. AI 에이전트가 이 브릿지를 통해 로컬 LLM을 호출합니다.

  • → LM Studio와 연결: qwen2.5-7b, llama-3.2-3b
  • → Ollama 폴백 자동 전환
  • → 스트리밍 + JSON-mode 지원
# 브릿지 상태 확인
curl http://127.0.0.1:4825/ping
# LM Studio 기본 포트
curl http://127.0.0.1:1234/v1/models
⚠ 단점

Anti-Gravity 창을 여러 개 열면 포트 4825 충돌이 납니다. 먼저 열린 창이 포트를 점유하고 두 번째 창은 브릿지 없이 실행됩니다. LM Studio가 꺼진 상태에서 AI를 호출하면 무한 대기 또는 에러가 납니다.

✓ 해결책

Anti-Gravity 창은 한 개만 유지하세요. AI 호출 전에 반드시 /ping으로 브릿지 상태를 확인합니다. LM Studio는 Anti-Gravity보다 먼저 실행하세요.

#3 커맨드 팔레트 & 단축키
Google ⌨

커맨드 팔레트

VS Code와 동일한 커맨드 팔레트를 씁니다. 모든 기능을 키보드만으로 실행 가능합니다.

  • 커맨드 팔레트 열기 Cmd/CtrlShiftP
  • 파일 빠른 열기 Cmd/CtrlP
  • 창 리로드 (확장 재시작) Cmd/CtrlShiftP
  • 터미널 열기 Ctrl`
  • 사이드바 토글 Cmd/CtrlB
⚠ 단점

일부 단축키가 OS 기본 단축키와 충돌합니다. Windows에서 Ctrl+Shift+P가 다른 앱과 겹치면 동작 안 합니다.

✓ 해결책

설정 → 키보드 단축키에서 직접 재매핑하세요. 또는 /keybindings-help 스킬로 Claude Code에서 키 설정을 도움받을 수 있습니다.

🔄

창 리로드 (Developer: Reload Window)

확장 프로그램 업데이트, 설정 변경, 브릿지 재시작 후 필수입니다. 전체 에디터를 다시 시작하지 않고 확장만 재초기화합니다.

# 커맨드 팔레트에서
Developer: Reload Window

# .vsix 재설치 후 순서
1. --install-extension *.vsix --force
2. Developer: Reload Window
3. curl :4825/ping # 브릿지 확인
⚠ 단점

리로드 후 열려 있던 터미널 세션이 모두 끊깁니다. 실행 중인 프로세스(로컬 서버 등)도 같이 죽습니다. 저장 안 된 파일이 있으면 날아갈 수 있습니다.

✓ 해결책

리로드 전 반드시 저장(Ctrl+S)하세요. 로컬 서버는 분리된 터미널(Windows Terminal 등)에서 실행하면 리로드에 영향 안 받습니다.

#4 통합 터미널
💻

통합 터미널

에디터 안에서 터미널을 바로 씁니다. 탭으로 여러 터미널을 관리하고, 분할 뷰로 에디터와 터미널을 동시에 볼 수 있습니다.

  • → npm / pip / git 명령 바로 실행
  • → 여러 터미널 탭 (서버, 테스트, git 분리)
  • → 터미널 출력 → 에디터 파일 연동
⚠ 단점

창 리로드 시 터미널 세션이 전부 죽습니다. 대용량 출력(빌드 로그 등)이 터미널에 쌓이면 에디터 전체가 느려집니다. Windows에서 PowerShell/CMD 인코딩 문제(한글 깨짐)가 나올 수 있습니다.

✓ 해결책

중요한 서버는 외부 터미널에서 실행하세요. 한글 깨짐은 chcp 65001 (UTF-8 설정) 으로 해결합니다. 빌드는 > build.log로 파일에 저장해서 터미널 버퍼 부하를 줄이세요.

🌿

Git 통합

사이드바 소스 컨트롤 패널에서 stage, commit, push를 GUI로 처리합니다. 브랜치 전환, diff 뷰, merge conflict 해결도 에디터 안에서 가능합니다.

  • → 변경 파일 한눈에 보기 + 스테이징
  • → 인라인 diff로 변경 내용 즉시 확인
  • → 브랜치 생성/전환 (하단 상태바 클릭)
⚠ 단점

복잡한 rebase, cherry-pick, stash는 GUI가 불편해서 결국 터미널로 가게 됩니다. 대형 레포(파일 수만 개)에서 git 상태 로딩이 느립니다.

✓ 해결책

기본 commit/push는 GUI 활용, 고급 git 작업은 터미널로 직접 처리하는 이분법이 효율적입니다. GitLens 확장을 추가하면 기능이 훨씬 보강됩니다.

#5 AI 사이드바 (Connect AI)
🤖

Connect AI 사이드바 + 가상 사무실

Anti-Gravity에 Connect AI 확장을 설치하면 사이드바에 AI 채팅 + 픽셀 아트 가상 사무실이 활성화됩니다. 9명의 AI 에이전트(CEO, 코다리, 이태원, 홍병길, 미향 등)가 로컬 LLM으로 돌아가며 실시간으로 작업합니다.

  • → 사이드바 채팅: "코다리야 게임 만들어줘" → 30초 내 결과물
  • → DISPATCH PROTOCOL 시각화 — 에이전트 이동 + 광선 + 파티클
  • → EZER AI 패널 → 키트 클릭 → 두뇌 자동 주입
  • → PayPal 매출 대시보드 실시간 연동
⚠ 단점

LM Studio가 꺼져 있거나 모델 로드가 안 된 상태에서 채팅하면 무한 대기 또는 타임아웃이 납니다. 가상 사무실 시각화(WebGL 애니메이션)가 무거워서 저사양 PC에서 에디터 전체가 느려집니다. 에이전트가 잘못 이해한 명령을 실행하면 원하지 않는 파일이 생성됩니다.

✓ 해결책

Anti-Gravity 실행 전에 LM Studio → 모델 로드 순서를 지킵니다. 시각화가 무거우면 가상 사무실 패널을 닫고 사이드바 채팅만 씁니다. 에이전트 실행 전 "어떻게 할 건지 먼저 설명해줘"로 계획을 확인하는 습관을 들이세요.

9명 에이전트 로컬 LLM 오프라인 동작 EZER AI 키트 PayPal 연동
#6 코드 편집 & IntelliSense
✏

IntelliSense & 자동완성

언어별 자동완성, 타입 추론, 오류 표시를 실시간으로 제공합니다. VS Code와 동일한 Language Server Protocol(LSP)를 사용합니다.

  • → TypeScript / JavaScript 타입 추론
  • → Python IntelliSense (Pylance)
  • → 인라인 에러 밑줄 + 빠른 수정 제안
⚠ 단점

언어 서버가 초기화될 때까지 수초 걸립니다. 대형 모노레포에서 타입 체크가 느리거나 멈춥니다. 일부 언어(특수 프레임워크)는 VS Code Extension이 없으면 IntelliSense가 없습니다.

✓ 해결책

파일 열고 5-10초 기다리면 IntelliSense가 살아납니다. 느리면 TypeScript: Restart TS Server 커맨드를 씁니다. 언어 서버가 없는 언어는 해당 확장을 .vsix로 설치하세요.

🐛

통합 디버거

중단점(breakpoint), 변수 감시, 콜스택 확인을 GUI로 처리합니다. Node.js, Python, 브라우저 디버깅을 지원합니다.

  • → F5로 디버그 실행
  • → 줄 번호 클릭으로 중단점 설정
  • → 변수 hover 시 현재 값 표시
⚠ 단점

launch.json 설정이 없으면 F5를 눌러도 실행이 안 됩니다. 복잡한 멀티프로세스 디버깅은 설정이 까다롭습니다.

✓ 해결책

처음 F5를 누르면 자동으로 launch.json 생성 안내가 뜹니다. 또는 "launch.json 만들어줘"라고 Claude Code에 요청하면 바로 만들어줍니다.

#7 확장 프로그램 관리
📦

확장 버전 관리

여러 버전의 .vsix가 extensions 폴더에 쌓이면 충돌납니다. 필요한 버전만 남기고 정리하는 관리가 필요합니다.

# 설치된 확장 목록 확인
ls ~/.antigravity/extensions/

# 구버전 삭제 (최신 버전만 남기기)
ls ~/.antigravity/extensions/ | grep connect-ai-lab
| grep -v "2.89.153"
| xargs -I{} rm -rf ~/.antigravity/extensions/{}
⚠ 단점

구버전을 삭제하지 않으면 어떤 버전이 로드됐는지 알 수 없습니다. extensions 폴더가 커질수록 Anti-Gravity 시작이 느려집니다.

✓ 해결책

새 버전 설치 후 구버전은 반드시 삭제하세요. 위 스크립트를 그대로 써서 최신 버전만 남기면 됩니다. 설치된 버전은 확장 패널 or 폴더 직접 확인이 가장 확실합니다.

⚙

settings.json 설정

VS Code와 동일한 settings.json으로 에디터 동작을 커스터마이즈합니다. 탭 크기, 테마, 언어별 설정 등 모든 옵션이 JSON으로 관리됩니다.

  • → 파일 자동 저장 (onFocusChange)
  • → 언어별 들여쓰기 (2/4 spaces)
  • → 터미널 기본 셸 설정
⚠ 단점

settings.json을 잘못 편집하면 에디터가 이상하게 동작합니다. VS Code와 Anti-Gravity의 설정 경로가 달라서 동기화가 안 됩니다.

✓ 해결책

커맨드 팔레트에서 Open Settings (JSON)으로 열면 자동완성이 됩니다. 설정 변경 전 백업해두세요. VS Code와 Anti-Gravity 양쪽에서 작업한다면 설정 파일을 공유 폴더에 두고 symlink로 연결합니다.

#8 멀티 창 & 분할 뷰
🪟

분할 편집기 (Split Editor)

하나의 창에서 파일을 좌우/상하로 분할해서 동시에 편집합니다. 프론트/백엔드 파일을 나란히 보면서 작업할 때 유용합니다.

  • → Ctrl+\ 로 현재 에디터 분할
  • → 최대 3-4열 분할 가능
  • → 같은 파일을 다른 위치에서 동시에 보기
⚠ 단점

Anti-Gravity 창 자체를 여러 개 띄우면 포트 4825 충돌. Connect AI 브릿지가 첫 번째 창에만 바인딩되어 두 번째 창에서는 AI가 동작 안 합니다. 분할 뷰를 너무 많이 열면 저사양 PC에서 버벅입니다.

✓ 해결책

여러 프로젝트는 창을 따로 띄우는 대신 하나의 창에서 분할 뷰로 해결하세요. 멀티 프로젝트가 꼭 필요하면 첫 번째 창(AI 있는 창)을 기본 작업 창으로 고정합니다.

🗂

워크스페이스 (.code-workspace)

여러 폴더를 하나의 워크스페이스로 묶어서 동시에 엽니다. 모노레포나 프론트+백엔드 분리 구조에서 유용합니다.

  • → 여러 루트 폴더를 한 창에서
  • → 워크스페이스별 settings.json 분리
  • → 최근 워크스페이스 빠른 전환
⚠ 단점

폴더가 많을수록 파일 탐색 트리가 복잡해져서 원하는 파일 찾기가 오히려 불편해집니다. 확장 일부가 멀티루트 워크스페이스를 제대로 지원 안 합니다.

✓ 해결책

2-3개 폴더 이내로 유지하세요. 더 많으면 분리된 Anti-Gravity 창으로 나누는 게 낫습니다 (AI가 필요 없는 창은 포트 충돌 무관).

🚀

Connect AI + Anti-Gravity 최적 실행 순서

1. LM Studio 실행 → 모델 로드 (qwen2.5-7b 권장)
2. Anti-Gravity 실행 → 프로젝트 폴더 열기
3. curl http://127.0.0.1:4825/ping 으로 브릿지 확인
4. 사이드바 채팅 → 에이전트 호출
이 순서를 지키면 연결 오류의 80%가 사라집니다.

⚠

Anti-Gravity vs VS Code — 언제 뭘 써야 하나

Anti-Gravity: Connect AI + 로컬 LLM + 오프라인 시연이 필요할 때. Google 생태계(Google AI, Gemini 연동)가 더 매끄럽습니다.
VS Code: 확장 호환성 문제가 생기거나, 특정 VS Code 전용 기능(Live Share, Remote SSH 등)이 필요할 때. Connect AI는 VS Code에서도 동일하게 설치됩니다.
