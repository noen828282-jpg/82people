# Node 7: Scene Brief Generation (장면 브리프 생성 노드)

## 1. 개요
* **목적**: 상품 정보와 이전 노드에서 벤치마킹한 경쟁 쇼츠의 연출 패턴을 토대로, 쇼츠 영상의 흐름을 씬별(Scene-by-scene) 비주얼 기획안과 사운드(내레이션) 연출 가이드로 상세화합니다.
* **트리거 방식**: 경쟁 쇼츠 패턴 분류 완료 시 단 건 실행.
* **주요 흐름**: 단일 상품 정보 수신 -> 15초 쇼츠 표준 구조(Hook-Demo-CTA)에 맞는 장면 분할 및 기획 구성 -> 각 장면의 시각적 및 청각적 연출 가이드 생성 -> `scene_briefs` 데이터 구조 매핑 -> 결과 저장 및 다음 노드로 전달.

## 2. 15초 쇼츠 표준 장면 구성안 (Hook-Demo-CTA)
영상의 몰입을 극대화하기 위해 다음과 같은 3단계 레이아웃 구조를 활용합니다.

1. **장면 1: Hook & 문제 제기 (0 ~ 3초)**
   * **목표**: 시청자의 시선을 3초 안에 사로잡음.
   * **비주얼**: 정리 안 된 어수선한 책상, 혹은 피로를 호소하는 유저의 클로즈업.
   * **사운드**: 긴장감 있거나 호기심을 유발하는 질문형 내레이션.
2. **장면 2: Demo & 상품 특징 시연 (3 ~ 12초)**
   * **목표**: 핵심 가치 제안 및 기능적 문제 해결 증명.
   * **비주얼**: 제품 언박싱, 깔끔한 책상 레이아웃, 제품 실사용 근접 시연(예: 타이밍, 스크롤링 등).
   * **사운드**: 긍정적이고 경쾌한 톤의 제품 기능 장점 소개.
3. **장면 3: CTA & 제휴 고지 (12 ~ 15초)**
   * **목표**: 제휴 링크 클릭 유도 및 법적 의무 고지.
   * **비주얼**: 제품 구매 링크가 가리켜지는 그래픽 또는 화면, 고지 자막 노출.
   * **사운드**: 명확한 혜택 안내 및 "아래 고정댓글 링크를 확인하세요" 문구.

## 3. 인터페이스 정의
* **입력 데이터**: `input_schema.json` 참조 (경쟁 분석 정보가 결합된 단일 상품 객체)
* **출력 데이터**: `output_schema.json` 참조 (각 장면의 기획서 `scene_briefs` 가 추가된 단일 상품 객체)

## 4. 실행 방법
```bash
python create_scene_brief.py --input sample_input.json --output output.json
```
