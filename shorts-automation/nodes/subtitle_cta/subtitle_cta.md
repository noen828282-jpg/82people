# Node 12: Subtitle/CTA Generation (자막 및 CTA 생성)

이 노드는 각 장면에 포함된 한국어 대사 스크립트를 시간에 맞춰 표준 SRT 자막 파일로 변환하여 물리 저장소에 기록하고, 쇼츠 비디오 화면에 합성될 배너 문구와 댓글 유도 화살표 등의 CTA(Call To Action) 그래픽 메타데이터를 기획하는 단계입니다.

## 1. 주요 기능 및 요구사항
1. **표준 SRT 자막 생성**:
   * 각 장면에 속하는 성우 내레이션 대사(`scenes_script`)와 지속 시간(`duration_seconds`)을 매핑합니다.
   * 첫 씬은 `00:00:00,000`에서 시작하며, 이후 씬은 이전 씬의 종료 시간부터 연이어 시작합니다.
   * 타임스탬프는 `HH:MM:SS,mmm` (시:분:초,밀리초) 규격의 문자열 포맷을 유지해야 합니다.
   * 자막 파일은 로컬 저장소(`nodes/assets/output/{prod_id}.srt`)에 UTF-8 인코딩으로 저장됩니다.
2. **CTA 오버레이 메타데이터 구성**:
   * **공정위 고지 배너**: 유튜브 쇼츠 및 인스타그램 릴스 규정에 맞게 화면 하단에 경제적 대가 고지 문구(Node 5에서 검증된 `disclosure`)를 상시 노출하기 위한 위치(Bottom, y-offset) 및 폰트 스타일 속성을 지정합니다.
   * **고정 댓글 링크 유도 화살표**: 영상의 마지막 장면(보통 제휴 링크를 안내하는 3막)에 시청자의 댓글창 클릭을 효과적으로 유도하기 위한 지시 그래픽 화살표(Arrow Overlay)의 애니메이션, 위치, 표시 타이밍(시작 시간 및 종료 시간) 메타데이터를 기획합니다.

## 2. SRT 타임스탬프 공식
* 누적 시작 시간 $T_{start}$ 및 누적 종료 시간 $T_{end}$ 계산:
  $$T_{start, i} = \sum_{j=1}^{i-1} Duration_j$$
  $$T_{end, i} = T_{start, i} + Duration_i$$
* 포맷팅: 소수점 이하 초 단위를 3자리 밀리초 문자열로 채우고, 시, 분, 초 역시 각 2자리 수 문자열로 변환합니다.
  * 예: `3.0초` $\rightarrow$ `00:00:00,000 --> 00:00:03,000`
  * 예: `9.0초` (누적 12초) $\rightarrow$ `00:00:03,000 --> 00:00:12,000`

## 3. CTA 그래픽 레이아웃 표준 (9:16 모바일 타겟)
* **공정위 배너**:
  * 위치: Bottom Center (y_position = 85%, 화면 하단부)
  * 색상: 흰색 텍스트, 반투명 검정 배경(Opacity 0.6)
  * 폰트 크기: 24px (1080x1920 해상도 기준)
* **화살표 애니메이션**:
  * 위치: Bottom Left (댓글 버튼 가리키는 포지션, x=80%, y=70%)
  * 노출 조건: 마지막 씬(3막) 재생 시작 후 종료 시까지 지속 표출
  * 효과: 바운싱(Bouncing) 애니메이션

## 4. 데이터 인터페이스

### 입력 데이터
* Node 11의 검수가 완료된 JSON 데이터를 수신합니다.

### 출력 데이터
* 물리적으로 생성된 `.srt` 파일의 절대 경로가 담긴 `subtitle_file_path` 필드가 추가됩니다.
* 자막의 디자인 요소 및 노출 타이밍 명세가 담긴 `cta_overlay` 객체가 추가됩니다.
