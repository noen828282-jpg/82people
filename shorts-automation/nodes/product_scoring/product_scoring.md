# Node 3: Product Scoring (상품 스코어링 노드)

## 1. 개요
* **목적**: 정제된 상품 목록을 평가하고 우선순위 점수(0~5점)를 계산하여, 어떤 상품의 쇼츠를 먼저 제작할지 결정합니다.
* **트리거 방식**: 상품 정제 완료 트리거 수신 시 자동 실행.
* **주요 흐름**: 정제된 상품 리스트 로드 -> 각 상품별 리뷰 긍부정 요약 및 스코어 계산 -> 스코어 내림차순 정렬 -> 결과 저장 및 다음 노드 전달.

## 2. 스코어 계산 공식 (Scoring Formula)
최종 스코어(Max 5.0)는 세 가지 가중치를 결합하여 계산합니다.
$$\text{Score} = (S \times 0.5) + (T \times 0.3) + (P \times 0.2)$$

1. **리뷰 평점 지표 ($S$, 5점 만점)**:
   * 상품 리뷰 텍스트 내에서 긍정 키워드(예: '편하다', '가볍다', '깔끔하다')와 부정 키워드(예: '불편하다', '무겁다', '아쉽다')의 비율을 산정하여 평가합니다.
2. **트렌드 지표 ($T$, 5점 만점)**:
   * 해당 상품 카테고리의 트렌드 가중치를 모사하여 지정합니다. (예: 1~5점 중 가중치 할당)
3. **가격 경쟁력 지표 ($P$, 5점 만점)**:
   * 10만 원 이하의 가성비가 높은 제품일수록 구매 허들이 낮아 쇼츠 판매에 유리하므로 더 높은 점수를 부여합니다.

## 3. 인터페이스 정의
* **입력 데이터**: `input_schema.json` 참조 (정제된 상품 정보에 리뷰 `reviews` 텍스트 리스트가 결합되어 들어옴)
* **출력 데이터**: `output_schema.json` 참조 (각 상품 객체에 `score` 및 `review_summary` 필드가 추가되고 정렬된 결과)

## 4. 실행 방법
```bash
python score_products.py --input sample_input.json --output output.json
```
