# Node 8: Script Generation (쇼츠 대본 생성 노드)

## 1. 개요
* **목적**: 기획된 각 장면의 연출 브리프(`scene_briefs`)를 활용해 성우 내레이션 또는 동영상 자막 리딩에 사용될 실제 구어체 한국어 대사 스크립트를 완성합니다.
* **트리거 방식**: 장면 브리프 생성 완료 시 단 건 실행.
* **주요 흐름**: 단일 상품 정보 및 장면 브리프 수신 -> 문장 단위 구어체 스크립트 변환 -> 전체 스크립트 글자 수 검증 (15초 러닝타임 범위) -> `scenes_script` 및 `overall_script` 데이터 결합 -> 결과 저장 및 다음 노드로 전달.

## 2. 대본 작성 및 분량 제약 규칙 (Script Rules)
쇼츠의 호흡과 가시성을 높이기 위해 다음 필터링/가공 기준을 적용합니다.

* **자연스러운 구어체 톤앤매너**:
  - 문어체("~다") 대신 대화형 구어체("~하셨죠?", "~해보세요", "~제품입니다")를 사용하여 친근감을 유도합니다.
* **글자 수 제약 (15초 표준 기준)**:
  - 1초당 읽을 수 있는 적정 한국어 글자 수는 공백 포함 **6자 ~ 7자** 내외입니다.
  - 따라서 **15초 전체 대본은 공백 포함 80자 ~ 100자 내외**로 작성해야 시청자가 어색하지 않게 소화할 수 있습니다.
  - 만약 생성된 전체 대본이 110자를 초과할 경우 경고를 띄우거나 축약 로직을 적용합니다.
* **필수 포함 요소**:
  - 상품명(`name`)의 주요 키워드 및 핵심 강점 1가지가 명확히 대본에 스며들도록 설계해야 합니다.

## 3. 인터페이스 정의
* **입력 데이터**: `input_schema.json` 참조 (장면 브리프가 완료된 단일 상품 객체)
* **출력 데이터**: `output_schema.json` 참조 (각 장면의 대사 및 전체 합본이 결합된 단일 상품 객체)

## 4. 실행 방법
```bash
python generate_script.py --input sample_input.json --output output.json
```
