# Node 6: Competitor Pattern Classification (경쟁 쇼츠 패턴 분류 노드)

## 1. 개요
* **목적**: 쇼츠 제작 대상 상품의 키워드 및 카테고리를 활용해 경쟁 플랫폼(YouTube Shorts, Instagram Reels 등)의 인기 동영상 연출 스타일을 분석하고, 최상의 성과를 낼 수 있는 비디오 제작 연출 포맷(패턴)을 추천합니다.
* **트리거 방식**: 제휴 링크 매핑이 완료된 상품 수신 시 단 건 실행.
* **주요 흐름**: 단일 상품 정보 수신 -> 해당 카테고리/키워드의 인기 경쟁 쇼츠 데이터 벤치마킹 모사 -> 훅(Hook) 문장 구성 방식 및 비주얼 연출 패턴(ASMR, Before/After 등) 도출 -> `competitor_analysis` 필드 추가 -> 다음 노드(장면 브리프 생성)로 전달.

## 2. 연출 패턴 및 훅 공식 정의
분석 엔진은 경쟁 쇼츠들의 특성을 아래와 같은 주요 카테고리로 분류합니다.

* **후킹(Hook) 스타일**:
  * `문제 제기형`: "아직도 지저분하게 줄 정리하시나요?" (공감 유도)
  * `경고형`: "손목 터널 증후군 막으려면 당장 치우세요!" (위기감 유도)
  * `직관적 비교형`: "1만 원짜리와 10만 원짜리 매트의 소음 차이" (비교 유도)
* **비주얼 연출 패턴**:
  * `ASMR 감성 타건`: 키보드 타건 소리와 연출 중심
  * `데스크테리어 셋업`: 공간 배치 및 깔끔함 중심
  * `실사용 기능 데모`: 연결 속도, 거치 안정성 등 기능 위주 시연

## 3. 인터페이스 정의
* **입력 데이터**: `input_schema.json` 참조 (제휴 링크 감사가 완료된 단일 상품 객체)
* **출력 데이터**: `output_schema.json` 참조 (경쟁 분석 정보 `competitor_analysis` 가 추가된 단일 상품 객체)

## 4. 실행 방법
```bash
python classify_patterns.py --input sample_input.json --output output.json
```
