# Node 4: Regulation Check (규제/표현 리스크 검사 노드)

## 1. 개요
* **목적**: 쇼츠 콘텐츠로 변환하기 전, 상품 정보에 포함된 텍스트 중 과장 광고 또는 허위 정보가 있는지 검사하여 법적 리스크를 미연에 방지합니다.
* **트리거 방식**: 상품 스코어링 완료 트리거 수신 시 자동 실행.
* **주요 흐름**: 스코어링된 상품 리스트 로드 -> 상품명 및 설명 텍스트 분석 -> 금지어 매칭 -> 리스크 결과(`risk: true/false`, `reason`) 마킹 -> 결과 저장 및 다음 노드 전달.

## 2. 금지어 및 심의 가이드라인 (Blacklist Guidelines)
국내 공정거래위원회 및 표시광고법의 가이드라인을 참조하여 아래 금지어 리스트를 적용합니다.

* **절대적/최상급 표현 (객관적 증명 불가)**:
  * `최고`, `완벽`, `가장 좋은`, `100% 보장`, `무조건`
* **즉각적 효과 암시 (과장성 주장)**:
  * `즉시 효과`, `바로 해소`, `단번에 해결`
* **안전성 오인 우려 표현 (비전기/전기 기구 공통)**:
  * `부작용 없음`, `인체 무해`, `100% 안전`
* **검증되지 않은 성능/효능 주장 (의료/치료 연상)**:
  * `치료`, `의학적 검증`, `완치`

## 3. 인터페이스 정의
* **입력 데이터**: `input_schema.json` 참조 (스코어링된 상품 배열)
* **출력 데이터**: `output_schema.json` 참조 (각 상품 객체에 `risk` 및 `reason` 필드가 결합된 배열)

## 4. 실행 방법
```bash
python check_regulations.py --input sample_input.json --output output.json
```
