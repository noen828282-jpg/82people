# Node 13: Upload Package Creation (업로드 패키지 빌드)

이 노드는 최종 완성된 미디어 자산, 자막 파일, 단축 제휴 링크, 해시태그 및 공정위 광고 문구가 포함된 업로드용 본문 텍스트를 규격화된 하나의 독립 패키지(Folder)로 빌드하고, 그 명세(Manifest)를 구성하는 단계입니다.

## 1. 주요 기능 및 요구사항
1. **패키지 디렉토리 구축**:
   * 각 상품별로 고유 폴더를 생성합니다. (예: `nodes/assets/output/{prod_id}_package/`)
   * 패키지 폴더 내부에 비디오 파일 및 자막 파일 등을 복사하거나 연계 저장하여 독립 배포가 가능한 구조를 지닙니다.
2. **매니페스트 파일 생성 (`manifest.json`)**:
   * 패키지 내부에 포함된 파일 구성(비디오 경로, 자막 경로 등)과 텍스트 정보(제목, 설명글, 해시태그)를 수록한 통합 명세서 JSON을 생성합니다.
3. **본문 텍스트 템플릿 엔진**:
   * 업로드용 설명글(Description)은 다음 요소가 개행과 함께 자연스럽게 결합된 구조를 지녀야 합니다.
     1. **헤드라인 제목**: 상품의 대표 키워드를 담은 호기심 유발형 제목 (예: "책상 위 혁명! [상품명] 솔직 타건 리뷰")
     2. **제휴 광고 링크**: 댓글 및 본문에 포함될 단축 파트너스 링크 (`affiliate_url`)
     3. **공정위 경제적 대가 고지 문구**: 투명하게 작성된 고지 문구 (`disclosure`)
     4. **태그 리스트**: 공백 구분된 해시태그 조합 (예: `#데스크테리어 #키보드추천 #[카테고리] #내돈내산 #쇼츠꿀템`)

## 2. 패키지 디렉토리 구조 예시
```text
nodes/assets/output/kb_001_package/
  ├─ manifest.json              # 업로드 명세 메타데이터
  ├─ kb_001.srt                 # 자막 파일
  ├─ kb_001_scene1.mp4          # 씬 1 비디오
  ├─ kb_001_scene2.mp4          # 씬 2 비디오
  └─ kb_001_scene3.mp4          # 씬 3 비디오
```

## 3. 해시태그 생성 규칙
* **필수 태그**: `#데스크테리어`, `#내돈내산`, `#쇼츠꿀템`
* **카테고리 매칭 태그**: 상품의 `category` 값에 따라 유동적으로 할당 (예: `keyboard` $\rightarrow$ `#키보드추천`, `#기계식키보드`)
* **브랜드/상품 태그**: 공백을 제거한 영어/한글 상품명 일부 (예: `#MXKeysMini`)

## 4. 데이터 인터페이스

### 입력 데이터
* Node 12의 SRT 및 CTA 설정 완료 데이터를 수신합니다.

### 출력 데이터
* 빌드 완료된 패키지의 절대 경로 `package_dir` 필드가 추가됩니다.
* 패키지 명세서의 파일 절대 경로 `manifest_file` 필드가 추가됩니다.
* 업로드 메타데이터가 담긴 `upload_package` 객체가 추가됩니다.
