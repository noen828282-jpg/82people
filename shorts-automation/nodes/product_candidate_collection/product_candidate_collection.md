# Node 1: Product Candidate Collection (상품 후보 수집 노드)

## 1. 개요
* **목적**: 쇼핑 쇼츠를 제작할 대상 상품 후보들을 외부 API(예: 쿠팡 파트너스 API 등) 혹은 웹 스크래핑을 통해 주기적으로 수집합니다.
* **트리거 방식**: Antigravity의 스케줄러를 활용한 주기적 타이머 실행 (예: 매일 오전 9시 실행)
* **주요 흐름**: 키워드 입력 받기 -> 제휴사 API 호출 / 크롤링 수행 -> 원본 상품 메타데이터 추출 -> JSON 파일 저장 및 다음 노드로 데이터 전달.

## 2. 인터페이스 정의
* **입력 데이터**: `input_schema.json` 참조 (수집 키워드 목록, 최대 수집 개수 등)
* **출력 데이터**: `output_schema.json` 참조 (수집된 상품 메타데이터 리스트 - ID, 상품명, 가격, 상세 URL, 카테고리, 이미지 링크 등)

## 3. 예외 및 실패 처리 (Failure Handling)
* **API 호출 실패 (네트워크 오류, 인증 토큰 만료 등)**: 최대 3회 재시도를 시도하며, 이후에도 실패할 경우 운영자 로그에 에러를 기록하고 휴면(idle) 상태로 전환합니다.
* **검색 결과 없음**: 키워드를 다양화하거나 백업 상품 데이터를 불러와 프로세스가 멈추지 않도록 조치합니다.

## 4. 실행 방법
```bash
python collect_products.py --input sample_input.json --output output.json
```
