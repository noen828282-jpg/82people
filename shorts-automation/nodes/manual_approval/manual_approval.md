# Node 14: Manual Approval (수동 검수 및 승인)

이 노드는 퍼블리싱 자동화의 최종 안전장치로서, 빌드된 업로드 패키지(영상, 자막, 매니페스트 설명글, 태그)를 관리자가 최종 점검하고 업로드를 최종 승인(Approve)하거나 반려(Reject)하는 단계입니다.

## 1. 주요 기능 및 요구사항
1. **수동 검수 화면 제공**:
   * CLI 상에서 빌드된 업로드 메타데이터(제목, 본문 글, 해시태그 목록, 파일 경로 목록)를 가독성 있게 출력합니다.
2. **체크리스트 문서 생성 및 관리**:
   * 검수 항목이 정리된 `approval_checklist.md` 마크다운 템플릿을 패키지 내부에 자동 제공하여 수동 점검 편의성을 돕습니다.
3. **인터랙티브 승인 피드백**:
   * CLI 사용자 입력을 지원하여 검수자가 직접 `[Y/N]` 혹은 승인 의견(`comment`)을 입력하게 처리할 수 있습니다.
   * 비대화형 자동화 환경(배치 실행)을 지원하기 위해 `--auto-approve` 옵션(또는 매개변수)으로 승인을 통과시키는 바이패스 모드도 함께 구현합니다.
4. **반려 시 워크플로우 처리**:
   * 반려 처리(`approved = false`)된 상품은 후속 배포 프로세스(API 연동 업로드)에서 배제되며, 리팩토링이나 재추천이 필요함을 알리는 상태 메타데이터를 추가합니다.

## 2. 수동 검수 체크리스트 세부 항목
* **영상 상태 검수**:
  - [ ] 씬1(훅)과 씬2(본문), 씬3(아웃트로)의 비주얼 전환이 매끄럽고 어색함이 없는가?
  - [ ] 영상 크기가 9:16 세로형 규격(1080x1920)을 충족하는가?
* **자막 상태 검수**:
  - [ ] 대사 텍스트에 오타나 어색한 비문이 없는가?
  - [ ] 자막 자막 싱크(SRT)가 오디오 내레이션 흐름과 알맞게 일치하는가?
  - [ ] 공정위 고지 자막 배너가 하단에 가독성 높게 표시되는가?
* **본문 및 링크 검수**:
  - [ ] 최저가 파트너스 링크가 클릭 가능하며 올바른 상품 페이지로 연동되는가?
  - [ ] 본문 내용 중 공정위 대가성 고지 문구가 누락 없이 온전히 기입되어 있는가?
  - [ ] 해시태그에 맞지 않는 키워드나 스팸성 단어가 섞여있지 않은가?

## 3. 데이터 인터페이스

### 입력 데이터
* Node 13의 업로드 패키지 빌드 완료 데이터를 수신합니다.

### 출력 데이터
* 승인 완료 정보가 담긴 `approval_status` 객체가 추가됩니다.
  - `approved`: 승인 성공 여부 (BOOLEAN)
  - `reviewer`: 검수자 식별 ID (STRING, 기본값: "admin")
  - `comment`: 승인 의견 또는 반려 사유 (STRING)
  - `approved_at`: 검수 완료 시간 (ISO 8601 타임스탬프)
  - `next_action`: 다음 조치 단계 ("publish" / "regenerate" / "hold")
