# Node 2: Product Refinement (상품 정제 노드)

## 1. 개요
* **목적**: 수집된 원본 상품 메타데이터를 정제하여 중복 제거, 필수 정보 누락 필터링, 그리고 타겟 카테고리에 맞는 상품만 선별하여 데이터 신뢰도를 높입니다.
* **트리거 방식**: 상품 수집 완료 데이터 수신 시 자동 실행.
* **주요 흐름**: 상품 목록 로드 -> 중복 및 누락 데이터 필터링 -> 정제된 상품 목록 저장 -> 다음 노드로 데이터 전달.

## 2. 정제 및 필터링 규칙 (Refinement Rules)
1. **중복 상품 제거**: 동일한 `id` 혹은 동일한 `name`을 가진 상품은 중복으로 판단하여 하나만 유지하고 제거합니다.
2. **필수 필드 검증**: `id`, `name`, `url` 필드가 누락되었거나 빈 값인 경우 제외합니다.
3. **품질 미달 제거 (Quality Cut)**:
   * 등록된 상품 이미지(`images`)가 1개도 없는 경우 제외 (비주얼 중심의 쇼츠 제작 불가).
   * 상세 설명(`description`)의 글자 수가 10자 미만으로 유의미한 정보가 없는 경우 제외.
   * 가격(`price`)이 누락되었거나 0 이하인 경우 제외.
4. **카테고리 매핑 및 검증**:
   * 데스크·모바일 생산성 카테고리(`keyboard`, `mouse`, `desk-accessory`, `charger` 등)만 허용하고, 패션/뷰티 등 범위 외 카테고리는 필터링합니다.

## 3. 인터페이스 정의
* **입력 데이터**: `input_schema.json` 참조 (수집된 원본 상품 배열)
* **출력 데이터**: `output_schema.json` 참조 (정제 및 필터링을 통과한 상품 배열)

## 4. 실행 방법
```bash
python refine_products.py --input sample_input.json --output output.json
```
