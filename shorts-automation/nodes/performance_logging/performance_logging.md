# Node 15: Performance Logging (퍼포먼스 지표 로깅)

이 노드는 퍼블리싱된 쇼츠 동영상의 노출 성과(조회수, 반응률) 및 실제 커머스 제휴 성과(링크 클릭수, 구매 전환수, 파트너스 수수료 수익) 데이터를 지능적으로 집계하고 히스토리 DB에 누적 기록하여 마케팅 효과를 평가하는 단계입니다.

## 1. 주요 기능 및 요구사항
1. **가상 성과 지표 계산 (시뮬레이션)**:
   * **기본 노출수 및 조회수**: 1,000 ~ 25,000회 범위에서 카테고리와 상품 점수(`score`) 가중치에 비례해 임의 생성합니다.
   * **반응(Engagement) 지표**:
     - 좋아요수: 조회수의 약 2% ~ 5%
     - 댓글수: 조회수의 약 0.1% ~ 0.5%
     - 공유수: 조회수의 약 0.05% ~ 0.2%
   * **링크 클릭수**: 조회수 대비 CTR(클릭률, 1% ~ 3% 내외)을 적용해 계산합니다.
   * **구매 전환수**: 클릭수 대비 전환율(CR, 0.5% ~ 1.5% 내외)을 적용해 계산합니다.
   * **수수료 수익 (Revenue)**: 쿠팡 파트너스 평균 수수료인 상품 가격(`price`)의 3% 및 구매 전환수 및 수량 가중치를 활용해 원화로 정밀하게 산출합니다.
2. **지표 누적 히스토리 데이터베이스 (`performance_history.json`)**:
   * 로컬 디렉토리 `nodes/assets/output/performance_history.json` 에 누적 데이터를 JSON Array 포맷으로 파일 저장합니다.
   * 이미 존재하는 상품 ID에 대해 성과 지표 수집 요청이 새로 유입될 경우, 기존 데이터에 누적치를 갱신하거나 혹은 타임라인 로그 형태로 히스토리를 순차 적재합니다.
3. **피드백 루프 지원**:
   * 본 노드에서 저장된 데이터는 마지막 노드인 Node 16(실험 추천 및 키워드 피드백)으로 전송되어, 성과가 저조한 상품군과 트렌드가 우수한 상품군을 지능적으로 분리 및 필터링하여 차기 수집 키워드를 제안하는 기초 분석 자료로 사용됩니다.

## 2. 수집 지표 명세

| 지표명 | 영문 필드명 | 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| 조회수 | `views` | Integer | 영상 재생 횟수 |
| 좋아요수 | `likes` | Integer | 긍정 반응 수 |
| 댓글수 | `comments` | Integer | 반응 댓글 수 |
| 공유수 | `shares` | Integer | 공유 횟수 |
| 링크 클릭수 | `clicks` | Integer | 고정댓글 단축링크 유입 클릭수 |
| 구매 전환수 | `conversions` | Integer | 클릭 후 실제 제품 구매 건수 |
| 제휴 수익금 | `revenue_krw` | Integer | 3% 수수료 기반 파트너스 누적 수익 |

## 3. 데이터 인터페이스

### 입력 데이터
* Node 14의 최종 수송 검수 승인이 완료된 JSON 데이터를 수신합니다. (승인 필드 `approval_status.approved`가 `true` 여야만 가상 성과 수집이 시작됩니다.)

### 출력 데이터
* 가상 측정되어 바인딩된 `performance_log` 객체가 추가됩니다.
