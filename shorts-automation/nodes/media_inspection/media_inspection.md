# Node 11: Media Inspection (미디어 검수)

이 노드는 비동기로 생성 완료된 쇼츠용 동영상 및 이미지 미디어 자산이 로컬 디스크 상에 정상적으로 존재하고 유효한 포맷과 규격을 충족하는지 자동 검수하는 단계입니다.

## 1. 주요 목적
* **존재성 검증**: 생성 큐에서 보고된 파일 경로에 물리 파일이 존재하는지 검증합니다.
* **규격 검증**: 파일의 크기(0바이트 초과 여부), 확장자(MP4, PNG), 비디오 비율(9:16 세로형) 및 권장 해상도(1080x1920)를 검증합니다.
* **오류 감지**: 미디어 생성 시 생성 지연이나 API 장애 등으로 비정상적으로 누락되거나 빈 파일로 남은 리소스를 조기에 확인하여, 불완전한 렌더링을 방지합니다.

## 2. 비즈니스 규칙 및 검수 기준
1. **물리 파일 확인**: 입력 객체 내 `media_assets`의 모든 `file_path`를 확인하여 로컬 디스크에 실제 파일이 있는지 검증합니다 (`os.path.exists`).
2. **크기 검증**: 파일 크기가 0바이트 초과이고 특정 제한치(예: 비디오의 경우 최소 1KB 이상)를 충족하는지 검증합니다.
3. **가상 속성 검사 (시뮬레이션)**:
   * **비디오 규격**: 비디오 해상도가 세로형 쇼츠 규격(`1080x1920`), 각 씬당 재생 시간(`duration_seconds`가 3.0 ~ 10.0초 사이)을 충족하는지 검증합니다.
   * **이미지 규격**: 이미지 해상도가 세로형 규격(`1080x1920`)을 충족하는지 가상 검증합니다.
4. **전체 결과 판정 (`inspection_passed`)**:
   * 검사 대상 리소스가 존재하지 않거나 결함이 있는 경우 `inspection_passed`는 `false`가 되며, 에러 정보가 `media_inspection_details`에 기록됩니다.
   * 모든 씬의 비디오와 이미지가 검수를 통과하면 `inspection_passed`는 `true`가 됩니다.

## 3. 데이터 인터페이스 규격

### 입력 스키마 (Input Schema)
* 이전 단계인 `media_generation_queue`의 출력 데이터 포맷을 그대로 유지합니다.
* 필수 포함 필드:
  * `id` (상품 고유 ID)
  * `media_assets` (동영상 및 이미지 파일 경로 객체)
    * `videos`: `scene_number`, `file_path`의 목록
    * `images`: `scene_number`, `file_path`의 목록

### 출력 스키마 (Output Schema)
* 검수 상세 이력과 검수 통과 여부 필드가 추가됩니다.
* 추가되는 필드:
  * `inspection_passed`: `true` / `false` (전체 검수 성공 여부)
  * `media_inspection_details`:
    * `videos`: 각 비디오 파일의 검수 상세 (파일명, 존재여부, 해상도, 재생시간, 크기, 검수 통과 여부)
    * `images`: 각 이미지 파일의 검수 상세 (파일명, 존재여부, 해상도, 크기, 검수 통과 여부)
